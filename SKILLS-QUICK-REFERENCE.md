## 📖 详细功能说明

### 1️⃣ problem-solver.md | 问题编排器 ⭐ 核心技能

**技能标识**：`problem-solver`

**核心功能**：
- 自动识别问题类型（纯发散/纯收敛/复合问题）
- 编排其他技能协同工作
- 输出完整解决方案

**典型场景**：
- 复杂技术问题（如何优化性能？）
- 架构设计（如何设计高可用系统？）
- 方案选择（如何改进代码质量？）

**激活提示词**：
```bash
# 直接触发
"如何提升前端项目性能"
"如何设计高可用系统"
"如何优化数据库查询"
"如何改进代码质量"
"如何提升开发效率"

# 明确指定
"完整解决方案：如何优化构建速度"
"先脑暴再辩论：如何降低成本"
```

**输出**：
```
1. 问题分类（A/B/C）
2. 发散阶段（20+ 方案清单）
3. 收敛阶段（最优方案选择）
4. 执行计划（分阶段实施）
```

**工作流程**：
```
用户提问
  ↓
problem-solver 识别问题类型
  ↓
如果是复合问题 C：
  → 调用 multi-thinker-brainstorm（发散）
  → 调用 multi-thinker-debate（收敛）
  → 输出完整解决方案
```

### 2️⃣ multi-thinker-brainstorm.md | 六维脑暴

**技能标识**：`multi-thinker-brainstorm`

**核心功能**：
- 6 个角色协同发散思维
- 产生创新点和可能性
- 跨领域知识融合

**6 个角色**：
- 🔭 哲学家：本质思考、价值观
- 🔬 科学家：可行性、数据支持
- 🎨 艺术家：用户体验、美学
- 💼 商业家：ROI、商业价值
- 🔧 工程师：实现细节、技术方案
- 🧠 心理学家：用户动机、行为模式

**典型场景**：
- 产品功能设计（设计一个支付功能）
- 用户体验优化（如何提升注册转化）
- 创新方案探索（智能手表能做什么）

**激活提示词**：
```bash
# 直接触发
"脑暴一下：智能手表的支付功能"
"6 个角色讨论：如何提升用户留存"
"启动脑暴模式：设计新功能"
"创意探索：社交应用的创新点"

# 上下文触发
"设计一个功能"
"如何改进"
"有什么创新点"
"用户体验优化"
```

**输出**：
- 20+ 创意方案
- 多维度视角分析
- 跨领域创新点

**不适用**：
- ❌ 明确决策问题（用 debate）
- ❌ 技术选型（用 debate）

### 3️⃣ multi-thinker-debate.md | 三方辩论

**技能标识**：`multi-thinker-debate`

**核心功能**：
- 3 个角色对抗思考
- 打破 AI 顺从性
- 做出平衡决策

**3 个角色**：
- 🚀 进取派（架构师）：追求极致、创新
- 🛡️ 怀疑派（SRE）：风险控制、安全
- ⚖️ 务实派（CTO）：成本收益、团队能力

**典型场景**：
- 架构决策（微服务 vs 单体）
- 技术选型（TypeScript vs JavaScript）
- 风险评估（这个方案会爆炸吗）

**激活提示词**：
```bash
# 直接触发
"三方辩论：是否采用微服务"
"多角色辩论：React vs Vue"
"批判性地分析这个架构"
"启动辩论模式：评估方案"

# 上下文触发
"评估方案 X"
"这个方案怎么样"
"哪个更好：A 还是 B"
"有风险吗"
"会不会过度设计"
```

**输出**：
- 3 个角色独立观点
- 交叉质询（互相攻击）
- 综合仲裁（平衡方案）
- 风险提示

**不适用**：
- ❌ 创意探索（用 brainstorm）
- ❌ 开放式问题（用 brainstorm）

## 💡 使用技巧

### 技巧 1：组合使用

```
问题解决完整流程：
  1. project-analyst（理解项目）
  2. problem-solver（制定方案）
     → brainstorm（发散）
     → debate（收敛）
  3. project-init（标准化）
  4. auto-qa（建立测试）
  5. git-master（版本控制）
```

### 技巧 2：问题类型判断

```
开放式问题 → brainstorm
  "如何改进用户体验？"

封闭式问题 → debate
  "是否采用 X？"

复杂问题 → problem-solver
  "如何优化 X？"
```

### 技巧 3：权限配置时机

```
新项目：
  npm init → setup-permissions → 使用技能

老项目：
  cd project → 技能自动提示 → 配置权限
```
